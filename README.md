# 🏦 ヘッジファンド仕様・究極のAI自律トレードエンジン (Zero-Base Audit Passed)

機関投資家レベルの**マルチファクター＆レジームスイッチング戦略**を搭載し、全14フェーズの「ゼロベース徹底監査」を完遂した、日本株完全自動売買システムです。

単なる売買スクリプトではなく、**異常検知・自己修復・取引所同期・リソース監視**までを備えた、「24時間365日の安定稼働」を前提とした運用プラットフォームへと進化しました。

## 💎 プロフェッショナル仕様への進化 (Zero-Base Audit)

本システムは、実弾運用におけるあらゆるリスクを徹底的に排除するため、「V1・V2プロフェッショナルコードレビュー」を経て独自の監視アーキテクチャや防衛層を構築しました。

1.  **ハイブリッド型・監視アーキテクチャ**: `yfinance` のデータ遅延（およびバージョン 1.2.0 以降の不安定性）を打破するため、銘柄選定は「朝の日足スキャン」、日中の監視は「カブコムAPIのリアルタイム時価情報（board）」を使い分けるハイブリッド方式を採用。
2.  **実効タイムスタンプ計層 (Effective Data Age)**: 単純な経過時間ではなく、市場の休業日や `jpholiday` を含む日本の祝日、さらに **年末年始休暇 (12/31〜1/3)** を考慮した「取引時間ベースの実効遅延」を計算。週末や祝日明けの誤動作を完全に排除。
3.  **スマート・ギャップ（窓開け）判定**: 朝9時の寄り付き値を監視。逆張り（RANGE）や急落（Gap Down）はリスクとして除外する一方、強気（BULL）時の上窓（Gap Up）はブレイクアウトの好機として許容する実戦的な判定ロジック。
4.  **カブコムAPI銘柄登録の動的管理**: `/register` APIの50銘柄制限を自動チャンク化で突破。監視対象を動的に登録・解除し、板情報のリアルタイム取得を最適化。
5.  **特別気配 (寄らず) への対応**: 9:00時点で値がつかない銘柄を即除外せず、初値がつくまで監視を「継続・待機」することで機会損失を防止。
6.  **yfinance完全停止へのフォールバック**: 外部データソース（yfinance）の深刻な遅延を検知した場合、Discord通知を行い運用を安全に自動休止。
7.  **注文ステート保護・取消追跡**: 注文がタイムアウトした際は確実にキャンセル要求を送信し、取引所側で「取消済」になるまでポーリング確認。キャンセル直後の約定による「ゾンビポジション」リスクをゼロ化。
8.  **株式分割自動同期**: API側のデータとローカル記録を照合し、分割・併合による株数と価格の乖離を自動調整。
9.  **分割利確（Scale-out）**: 利益がATRの1.5倍に達した時点で半分を利確。残り半分はトレール幅を広げて特大トレンドを狙うプロップファーム仕様。

## 🔥 搭載ロジック

### 1. レジーム（相場環境）判定
日経平均のデータから現在の相場環境を自動判定。
- **BULL（強気）**: **「押し目買い（Pullback）」戦略**。SMA50による中期トレンド確認後、VWAP付近まで一時的に押した「一番美味しいポイント」を狙撃。高値掴み（VWAPから+3%以上）を物理的に排除。
- **RANGE（もみ合い）**: ミーンリバージョン戦略（逆張り）で安値を拾う。**（※「落ちるナイフ」を回避するため、直近15分足の陽線反発を必須条件とするモメンタムフィルター搭載）**
- **BEAR（パニック）**: **新規買付を物理的にロック**し、保有ポジションの出口戦略のみ実行。

### 2. 数学的スクリーニング & AI判定
- **数学フィルター**: 4000銘柄を瞬時にスキャンし、出来高・MACD・RVI・VWAP等から上位候補を抽出。**1日平均の最低売買代金を10億円に設定**し、流動性枯渇によるデッドマネー化を排除。
- **AI防護壁**: 数学フィルターを通過した「スコア最上位（1位）の銘柄」に対してのみ、Gemini 1.5 Flash（またはGroq）が悪材料ニュースを最終審議し、安全であれば高速に発注（M-2最適化済み）。

## 🚫 インサイダー取引防止設定

プロジェクトルートの `insider_exclusion.json` に銘柄コードを追加すると、そのコードは**新規買付スキャンから完全除外**されます。

```json
{
  "_comment": "インサイダー取引防止リスト",
  "codes": [
    "7203",
    "9984"
  ],
  "notes": {
    "7203": "勤務先",
    "9984": "取引先"
  }
}
```

**動作仕様:**
- ✅ リストに追加した銘柄は、新規買付候補として絶対に選択されません  
- ✅ **既に保有しているポジションの損切り・利確には影響しません**（売れなくなる事態を防ぐため）  
- ✅ ファイルを削除・空にしても、エラーなく通常動作します  
- ✅ ボット起動中に `codes` リストを変更した場合、次のスキャンサイクルから自動で反映されます

## ⚠️ 本番運用 (KABUCOM_LIVE) 時の重要注意点

本システムでリアルトレード（自己資金による自動売買）を行う場合、以下の制約とリスクを事前にご認識ください。

### 1. auカブコム証券の「信用取引口座」の開設が必須です
本システムは取引コストの利益圧迫（エッジの破壊）を防ぎ、完全な「取引手数料0円・金利0円」を実現するために、API経由で**「デイトレード信用（一般信用ワンデイ）」**を用いた発注を行います。
現物口座しか開設されていない場合は、APIが注文をリジェクトします。必ず事前に信用口座の開設と、必要な委託保証金の入金を行ってください。

### 2. ストップ安（特別気配）による「翌日持ち越しリスク」
15:15の市場引け直前に、本ボットはシステム上の原則として「保有中の全ポジションを成行で強制決済」します。
しかし、対象銘柄がストップ安張り付き等で**完全に買い手が不在（出来高0）の場合、証券取引所の仕様上、どんなプログラムを使っても物理的に約定しません。**
結果としてシステムはポジションを「取消済」とするか「翌日にロールオーバー」することになります。この際生じる翌日のギャップダウンによる損失は、自己責任許容範囲（システムトレードの仕様限界）として運用してください。（※このリスクを最小化するため、売買代金10億円以上の流動性フィルターを持たせています）

### 3. APIパスワードとインサイダー制約
API通信には`.env`での `KABUCOM_API_PASSWORD` が必要です。パスワード更新時は必ず`.env`も書き換えてください。
また、ご自身の勤務先・取引先等、インサイダーの疑いをかけられる銘柄については、運用開始前に必ず `insider_exclusion.json` に手動で登録してください。
