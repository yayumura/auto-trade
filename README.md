単なる売買スクリプトではなく、**異常検知・自己修復・取引所同期・リソース監視**までを備えた、「実運用での絶対的な安定」を追求した次世代型トレードエンジンへと進化しました。

## 💎 プロフェッショナル監査による進化 (Professional Audit Outcomes)

実弾運用における壊滅的なリスクを排除するため、以下の高度なインフラ基盤を構築済みです。

1.  **OMS（注文管理システム）の高度化**: 最良気配への「追従発注 (Chase Order)」と、一定時間未約定時の「強制執行」を組み合わせたプロ仕様の執行エンジン。
2.  **ハイブリッド型・監視アーキテクチャ**: 銘柄選定は「朝の日足スキャン」、日中の監視は「カブコムAPIのリアルタイム15分足バッファ」を使い分けるハイブリッド方式。yfinanceの遅延問題を完全に克服。
3.  **実効タイムスタンプ判定 (Effective Data Age)**: 単純な経過時間ではなく、市場の休業日や年末年始休暇を含む「取引時間ベースの実効遅延」を計算。
4.  **インテリジェント・AIフィルタ**: 非同期・フェイルオーバー構造を採用。AI APIの障害時も「一時承認」でシステムを止めず、かつ詳細な理由をログに記録。
5.  **2024年最新呼値ルール**: 東証の最新呼値テーブルに基づき、価格帯を跨ぐ際も常に有効なティックサイズでアトミックに発注。
6.  **状態永続性 (Persistence)**: 買付・売却・一部約定のあらゆる局面でポートフォリオとウォッチリストを即時保存し、再起動時の不整合をゼロ化。
7.  **分割利確（Scale-out）**: 利益がATRの1.5倍に達した時点で半分を利確。残り半分はトレール幅を広げて特大トレンドを狙うプロップファーム仕様。

## 🔥 戦略ロジック

### 1. レジーム（地合い）スイッチング
日経平均のデータをリアルタイム解析し、戦略を動的に切り替え。
- **BULL（強気）**: **「押し目買い（Pullback）」戦略**。SMA50によるトレンド確認後、VWAP付近の押し目を狙撃。高値掴みを排除。
- **RANGE（もみ合い）**: **逆張りミーンリバージョン**。ボリンジャーバンド下限かつRSI低位株を狙う。※直近足の陽線反発を必須条件とする momentum filter 搭載。
- **BEAR（パニック）**: **新規買付を100%ロック**。出口戦略のみ実行。

### 2. 数学的スクリーニング & AI判定
- **数理フィルタ**: 4000銘柄をスキャンし、売買代金10億円以上の流動性上位から抽出。
- **AI防護壁**: 数学フィルターを通過した銘柄に対し、LLM（Gemini 2.0 Flash/OpenAI等）が悪材料ニュースを最終審議。

## 🚀 セットアップと実行

### 1. 環境構築
```bash
pip install -r requirements.txt
```

### 2. 環境変数 (.env)
```env
GEMINI_API_KEY=your_key
OPENAI_API_KEY=your_key
DISCORD_WEBHOOK_URL=your_webhook
# API利用件のためのパスワード (kabuステーションのAPI設定で決めたもの)
KABUCOM_API_PASSWORD=your_api_password
# アプリ起動・ログインのためのパスワード (アプリを立ち上げる時に入力するもの)
KABUCOM_LOGIN_PASSWORD=your_app_password
KABUCOM_ACCOUNT_TYPE=4 # 4:特定口座 (デフォルト), 2:一般口座
TRADE_MODE=SIMULATION # または KABUCOM_TEST / KABUCOM_LIVE
```

### 3. 取引モードの使い分け

本システムには「過去検証（バックテスト）」と「リアルタイム運用」の2つのシミュレーション環境があります。

#### A. 過去検証 (バックテスト / Backtest)
過去のチャートデータ（15分足）を使用して、ロジックの有効性を短時間で検証します。
```bash
# 過去20日間のデータでシミュレーション実行
python backtest.py
```
*   **注意**: `backtest.py` は常に **「シミュレーションモード用の執行ロジック」** で動作します。
    *   **共通ロジック**: `core/logic.py` にある「売買判断（いつ買う・いつ売るか）」の判断基準は、本番・テスト・バックテストですべて共通です。
    - **執行方式**: バックテストでは API を介さず、`SimulationBroker` を使用して仮想口座内で即時約定・仮想手数料計算を行います。このため、`.env` の `TRADE_MODE` が何であっても、バックテストの結果が実口座に影響を与えることはありません。

#### B. リアルタイム運用 (シミュレーション・本番)
実際の市場時間に合わせて動作し、現在の板状況やトレンドをリアルタイム監視します。
```bash
# 平日の朝 08:50 に起動してください（15:30に自動終了）
python auto_trade.py
```
*   **SIMULATIONモード**: リアルタイムの仮想取引（ペーパートレード）。
*   **KABU_LIVE/TESTモード**: 実際の証券口座への発注。

## 🛡️ 究極の安全機構
- ✅ **寄り付き30分回避**: 9:30までの混沌とした値動きへのエントリーを禁止。
- ✅ **スマート・ギャップ判定**: 窓開けを監視し、逆張り時の急落や強気時の窓開けを適切にフィルタ。
- ✅ **インスタンスロック (Singleton Guard)**: `.bot_sim.lock` による多重起動の物理的遮断。
- ✅ **kabuステーション自動起動・ログイン**: アプリの立ち上げからログインまでを自動化。OTP（ワンタイムパスワード）が必要な場合はDiscordへ通知し、手動入力を待機するハイブリッド仕様。
- ✅ **APIサーバー自動検知 (Port 18080/18081)**: ログイン完了後、APIサーバーが起動したことを自動で検知してトレードを開始。
- ✅ **特別気配・ゼロ除算ガード**: 寄らずの銘柄や価格異常時のエラーを物理的にゼロ化。
- ✅ **注文滞留オートキャンセル**: 執行中に5分以上約定しない注文を自動検知して取り消し、資金のロックを防止。
- ✅ **APIレートリミット保護**: リクエスト予算管理によりIP制限を自動回避。
- ✅ **Unicode/絵文字エラー対策**: Windows(CP932)環境での実行エラーを回避するため、出力をプレーンテキスト化済み。

## ⚠️ 本番運用 (KABUCOM_LIVE) 時の重要注意点

1. **信用取引口座の開設**: 手数料0円の「デイトレード信用」を使用するため、信用口座が必須です。
2. **管理者権限での実行**: GUI自動操作（pywinauto）を使用するため、ターミナルを「管理者として実行」して起動する必要があります。
3. **翌日持ち越しリスク**: ストップ安張り付き等で買い手不在の場合、15:15の決済注文が約定せず翌日に持ち越される可能性があります。（※流動性フィルターでリスクを低減済み）
4. **インサイダー防止**: insider_exclusion.json に銘柄を追加し、新規買付から除外してください。
