# 🏦 ヘッジファンド仕様・AIアルゴリズムBOT（東証シミュレーター）

機関投資家レベルの**マルチファクター＆レジームスイッチング戦略**を実装した、完全自律型の日本株シミュレーショントレードBOTです。

東証の全銘柄（約4000銘柄）を15分足で高速数学スクリーニングし、相場環境（レジーム）に応じて**モメンタム戦略**と**ミーンリバージョン戦略**を自動切替。最終フィルターとしてAI（Gemini / Groq）が悪材料ニュースをチェックし、安全な銘柄のみに絞り込んで仮想売買を行います。

## 🔥 搭載ロジック

### ステップ0：営業時間フィルター
- 平日の東証取引時間（**9:00〜11:30 / 12:30〜15:30**）のみ稼働
- 土日・祝日・営業時間外は即終了し、APIを無駄に消費しない

### ステップ1：レジーム（相場環境）判定
日経平均（^N225）の過去1ヶ月の日足データ（Adj Close）から現在の相場環境を自動判定：

| 条件 | レジーム | 行動 |
|:---|:---|:---|
| 現在値 > SMA20（20日移動平均） | **BULL（強気）** | モメンタム戦略で攻める |
| SMA20の-5%以内 | **RANGE（もみ合い）** | 逆張り戦略で拾う |
| SMA20の-5%以下 or 年率ボラ30%超 | **BEAR（パニック）** | 新規買い完全停止・保有のみ管理 |

### ステップ2：保有ポジション管理（利確・損切り・タイムストップ）
保有中の全銘柄について15分足データを取得し、以下の優先順位で決済を判定：

1. **タイムストップ**（15:00以降） → オーバーナイトリスク回避のため強制決済（大引け15:30の30分前）
2. **絶対損切り** → 現在値 ≤ 買値 − ATR×2.0
3. **トレール利確** → 現在値 ≤ 最高値 − ATR×1.5（かつ買値+0.5%以上で利確）
4. **建値撤退** → 利益がほぼない状態での撤退

**安全機構：**
- 株式分割・配当落ちによる疑似暴落を**分割係数（Adj Close / Close）**で動的補正
- 売却時は現在値の**0.1%不利**にスリッページをシミュレーション
- 利益確定時は**税率20.315%**を自動控除

### ステップ3：新規エントリーの前提チェック
- 最大4銘柄保有中 → スキャン中止（リスク分散上限）
- 9:30以前 → スキャン中止（寄り付きのランダムウォーク回避）
- 14:30以降 → スキャン中止（引け前の新規買いは即決済リスク）

### ステップ4：数学的スクリーニング（約4000銘柄 → 上位3銘柄）
全銘柄を**500銘柄ずつチャンク分割**してyfinanceから15分足5日分を安定取得。

#### 共通フィルター（全レジーム共通）
- 株価100円未満 → 除外
- **1日平均売買代金 3億円未満** → 除外（スリッページ防止）

#### BULL戦略（モメンタム）
| フィルター | 条件 |
|:---|:---|
| 出来高急増（RVOL） | 100本移動平均の**2.5倍以上** |
| MACDクロス | MACD > Signal |
| RSI過熱防止 | RSI < 75（イナゴタワー回避） |
| VWAP超え | 当日Typical Price VWAPを上回る |

スコア = `(出来高倍率×10) + (RSIの上昇幅)`

#### RANGE戦略（ミーンリバージョン）
| フィルター | 条件 |
|:---|:---|
| SMA下抜け | 終値 < SMA20 |
| ボリンジャー下限割れ | 終値 < BB Lower(-2σ) |
| 売られすぎ | RSI < 35 |

スコア = `(乖離率の深さ×10) + (出来高倍率×5)`

### ステップ5：AI定性フィルター（Gemini API）
スコア上位3銘柄を順にチェック：
1. Google News RSSで最新ニュース5件を取得
2. **Gemini 2.5 Flash** に「致命的な悪材料（下方修正・不祥事・上場廃止等）があるか？」を質問
3. Gemini API制限時は **Groq（Llama 3.3 70B）** に自動フォールバック
4. 最初に合格した1銘柄のみを買付候補に

### ステップ6：ポジションサイジング＆買付
- **ボラティリティベースの株数計算**：1トレードのリスク = 総資産の2%
- 日本株の**単元株（100株単位）**に強制丸め込み
- 購入時は現在値の**0.1%高く**スリッページをシミュレーション
- NaN・異常値検知時は買付を強制キャンセル
- 残高マイナス防止のフェイルセーフ搭載

## 🚀 セットアップ

### 1. 必要なライブラリのインストール
```bash
pip install yfinance pandas numpy google-genai feedparser groq python-dotenv requests flask
```

### 2. 環境変数の設定
プロジェクトのルートに `.env` ファイルを作成し、以下を記入：
```
GEMINI_API_KEY=your_gemini_api_key
GROQ_API_KEY=your_groq_api_key
DISCORD_WEBHOOK_URL=your_discord_webhook_url
```

### 3. 銘柄リストの配置
[JPXの上場銘柄一覧](https://www.jpx.co.jp/markets/statistics-equities/misc/01.html)から `data_j.csv` をダウンロードし、プロジェクトルートに配置。

### 4. 実行
```bash
# トレードBOT
python auto_trade.py

# ダッシュボード（別ターミナル）
python dashboard.py
```

### 5. 定期実行（推奨）
Windowsタスクスケジューラーで **30分間隔** で自動実行を設定。

推奨起動時刻（営業日のみ）：
```
前場:  9:30, 10:00, 10:30, 11:00, 11:20
後場: 12:30, 13:00, 13:30, 14:00, 14:30, 15:00
```

> ⚠️ 15分間隔は非推奨です。15分足の未確定データでダマシが増え、API消費も倍増します。

## ⚙️ 主要パラメータ

| パラメータ | 値 | 説明 |
|:---|:---|:---|
| `INITIAL_CASH` | 1,000,000円 | シミュレーション初期資金 |
| `MAX_POSITIONS` | 4 | 最大同時保有銘柄数 |
| `MAX_RISK_PER_TRADE` | 2% | 1トレードあたりの最大リスク（対総資産） |
| `ATR_STOP_LOSS` | 2.0 | 損切りライン（ATRの倍数） |
| `ATR_TRAIL` | 1.5 | トレールストップ（ATRの倍数） |
| `TAX_RATE` | 20.315% | 利益に対する税率 |
