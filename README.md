## 🛡️ 運用信頼性 (Resilience & Safety)

本番運用を「手放し」で行うための、高度なエラーハンドリング機能を標準搭載しています。

### 1. 停電・クラッシュからの自動復帰
プログラムが不意の停電などで停止しても、再起動時に **kabuステーション API から現在の保有残高・口座残高を自動取得** して状態を復元します。以前の記憶に頼らず「事実」から復元するため、即座に監視を再開できます。

### 2. 異常データ検知：Gap Guard
yfinance のデータ取得誤り（株式分割・権利落ちの反映漏れなど）を自動検知。前日終値と当日価格の乖離が規定値を超える銘柄を「汚染データ」として自動スキップし、誤った注文を防ぎます。

### 3. スタック注文の自動キャンセル
未約定のまま **5分以上放置された注文を自動キャンセル** します。market impact を避け、資金を次の機会に向けます。

### 4. スマート・シャットダウン
- **15:30（大引け）**: 全ての未約定注文をキャンセルし、kabuステーションを自動終了します。
- **祝日・土日**: 起動時に自動判定してkabuステーションを閉じ安全に待機します。

### 5. インスタンス多重起動防止
ロックファイル（`.bot_sim.lock`）による排他制御。既に別プロセスが動いている場合は起動をブロックし、二重注文を防ぎます。古いプロセスが異常終了した場合は PID確認の上、自動でロックを解除します。

## ⚡ V12.0 戦略コンセプト (Growth Monster)

### エントリー条件（全5つ満たした銘柄のみ）

| 条件 | 内容 |
|:---|:---|
| SMA20上 | 終値 > 20日移動平均（短期トレンド確認） |
| Ret3 > 5% | 直近3日間で5%以上上昇（モメンタム確認） |
| 出来高急増 | 出来高 > 20日平均出来高 × 1.5倍 |

| 価格フィルター | 200円 〜 10,000円（流動性確保） |

### エグジット条件（優先順）

1. **利確（TP=5%）** → 高値が買値×1.05 以上に達したら即時売却
2. **損切り（SL=3%）** → 安値が買値×0.97 以下に達したら即時売却
3. **タイムリミット（T=3日）** → 3日保有後も決着つかない場合 **翌朝始値で売却**

### ポジション管理

- 最大保有銘柄数：**5銘柄**（均等配分）
- 銘柄選定：Ret3（3日騰落率）の高い順に優先
- 本番はATRベースのリスク管理でロットを調整

## 📊 ツール使用方法

### 1. バックテスト（現在の設定で詳細検証）

```bash
python backtest.py
```

- `core/config.py` のパラメータで実行
- NET（税引後）と GROSS（税引前）の両方を比較表示
- 月次資産推移テーブルを出力

### 2. パラメータ最適化（グリッドサーチ）

```bash
python optimizer.py
```

- TP / SL / POS / TIME の全組み合わせを高速探索
- NumPyベクトル演算で高速化済み
- NET利益率でランキング表示

### 3. ロボットの起動

`run_bot.bat` をダブルクリックするか、Windows タスクスケジューラに登録して実行します。

- **実行ファイル**: `run_bot.bat`
- **ログの確認**:
  - **全体ログ**: `data/kabucom_test/logs/task_scheduler.log`（起動・停止の記録。5MB超えで自動リセット）
  - **日別詳細**: `data/kabucom_test/logs/console_YYYY-MM-DD.log`（スキャン内容・売買理由の詳細。30日で自動クリーンアップ）


`data/symbols_with_market.csv` に市場区分付き銘柄リストが必要。

### 4. `run_bot.bat` のPythonパス設定

```bat
SET PYTHON_EXE=C:\path\to\python.exe  ← 自分の環境に合わせて変更
```

> [!TIP]
> `TRADE_MODE=SIM` でローカルシミュレーション、`KABUCOM_TEST` で証券会社検証環境、`KABUCOM_LIVE` で本番取引ができます。最初は必ず `SIM` または `KABUCOM_TEST` で動作確認してください。

## 🤖 本番ボットの動作フロー

```
起動
 ↓ kabuステーション自動起動・API接続確認
 ↓ 起動前点検（preflight check）
 ↓ [ループ 30秒毎]
    ├─ 市場レジーム判定（BULL / RANGE / BEAR）
    ├─ ポジション管理（TP / SL / タイムリミット判定）
    │    └─ タイムリミット到達 → 翌朝始値で売却（next_open）
    ├─ 銘柄スキャン（9:30〜14:00）
    │    ├─ V12テクニカルフィルター
    │    ├─ AIニュース定性フィルター
    │    └─ 注文執行（成行・寄付き）
    └─ Discord通知・ログ記録
 ↓ 15:30 大引け → kabuステーション終了・ボット停止
```

> [!NOTE]
> BEAR相場（日経225が50日線・200日線を共に下回る）の場合、新規買いを停止してリスクを回避します。
